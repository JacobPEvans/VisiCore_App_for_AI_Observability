# VisiCore App for AI Observability

AI observability for Claude Code, Gemini CLI, and other LLM providers. Two Splunk packages providing dashboards and knowledge objects aligned with OpenTelemetry GenAI semantic conventions.

## Architecture

```
Filesystem -> Cribl Edge -> Cribl Stream -> Splunk HEC -> Splunk Enterprise
   (JSON)      (packs)      (routing)       (port 8088)    (index=claude)
```

## Packages

### VisiCore_App_for_AI_Observability

Dashboards (Dashboard Studio v2):

- **Overview** - High-level KPIs, token trends, model distribution, tool usage, and session activity
- **Token Usage** - Detailed token consumption analysis across input, output, cache read, and cache creation
- **Cost Analysis** - Dollar cost estimation by model, session, and project with cumulative trending
- **Tool Activity** - Tool call patterns, file operations, error rates, and tool-level drill-downs
- **Sessions** - Session-level analysis with duration, token depth, and project attribution
- **Cache Performance** - Prompt caching efficiency, cache hit rates, and estimated token savings

### VisiCore_TA_AI_Observability

Knowledge objects:

- Field extractions (props.conf, transforms.conf)
- OpenTelemetry GenAI field aliases
- Search macros for common patterns
- Lookups for model pricing and classification
- Eventtypes and tags for CIM compliance

## Token Model

Aligned with [ccusage](https://github.com/ryoppippi/ccusage). Four token types:

| Token Type | Field | Description |
|---|---|---|
| Input | `input_tokens` | Tokens sent to the model |
| Output | `output_tokens` | Tokens generated by the model |
| Cache Read | `cache_read_input_tokens` | Tokens read from prompt cache |
| Cache Creation | `cache_creation_input_tokens` | Tokens written to prompt cache |

Total = input + output + cache_read + cache_creation. Deduplication via `messageId:requestId` composite key.

## OTel Field Mapping

| Native Field | OTel Convention |
|---|---|
| message.model | gen_ai.response.model |
| sessionId | gen_ai.conversation.id |
| message.usage.input_tokens | gen_ai.usage.input_tokens |
| message.usage.output_tokens | gen_ai.usage.output_tokens |
| message.stop_reason | gen_ai.response.finish_reasons |
| "anthropic" | gen_ai.provider.name |

## CIM Data Model Mapping

| Native Field | CIM Field | Data Model |
|---|---|---|
| type | action | Web, Change |
| message.model | dest | Web |
| sessionId | session_id | Web |
| "anthropic" | vendor | All |
| "claude_code" | product | All |
| tool name | object, command | Change |
| input.file_path | object_path | Change |

## Installation

```bash
# Package both apps
./scripts/package.sh

# Install on Splunk (TA first, then App)
splunk install app build/VisiCore_TA_AI_Observability-*.tar.gz
splunk install app build/VisiCore_App_for_AI_Observability-*.tar.gz
splunk restart
```

1. Install the TA (`VisiCore_TA_AI_Observability`) first, then the App
2. Ensure indexes exist: `claude`, `gemini`
3. Configure data inputs via Cribl Edge packs

## Usage

Navigate to the VisiCore app in Splunk Web. Six dashboards are available:

1. **Overview** - Start here for a summary of all AI activity
2. **Token Usage** - Drill into token consumption patterns
3. **Cost Analysis** - Track spend by model, session, and project
4. **Tool Activity** - Monitor tool call patterns and file operations
5. **Sessions** - Analyze individual coding sessions
6. **Cache Performance** - Evaluate prompt caching efficiency and savings

All dashboards support a time range picker and use the base search + extend pattern for efficient search execution.

./scripts/package.sh
